🎯 Same Metadata, Different Component, Infinite Possibilities

  • Same Data Source: Identical metadata fields as the card component example
  • Different Template: Hero banner layout instead of card layout
  • Same Slots: Still using the universal 4-slot pattern
  • Different Mapping: Metadata mapped differently for hero context (title becomes hero headline, subtitle becomes supporting text)
  • Infinite Variations: Could also be: compact banner, video hero, split hero, carousel hero, etc.
  • Zero Duplicate Code: One metadata structure → many component presentations
